# Web UI

## Why the web UI exists

The web UI makes the Enterprise Data Strategy Agent easier to demonstrate to stakeholders who may not want to run CLI commands. It presents the same open-source strategy engine in a browser so teams can discuss metadata quality, reporting trust, governance gaps, and remediation priorities interactively.

## Who it is for

The first audience is enterprise data managers, analytics leaders, Domo admins, BI platform owners, data governance leads, and non-technical executives who need a readable summary of metadata risk and remediation options.

## How to run it

Install dependencies from the repository root:

```bash
python -m pip install -r requirements.txt
```

Launch Streamlit:

```bash
streamlit run app.py
```

## Use sample metadata

In the sidebar, choose **Bundled sample inventory**. The app loads `data/sample_domo_inventory.json`, which is synthetic Domo-style metadata containing datasets, dashboards/cards, owners, business domains, certification flags, sensitivity levels, refresh information, and usage signals.

## Upload custom metadata

Choose **Upload custom JSON inventory** in the sidebar and upload a JSON file that follows the same inventory shape as the bundled sample. The app validates the upload and shows a friendly error when the JSON is malformed or missing required inventory fields.

## Apply a policy

The policy selector supports three modes:

- **Default policy** — built-in assumptions for scoring, freshness thresholds, severities, and stakeholder roles.
- **Bundled sample policy** — loads `config/sample_strategy_policy.yml`.
- **Upload custom YAML policy** — validates an uploaded policy file with the same loader used by the CLI.

Policy context matters because industry, maturity stage, owner role, platform, scoring weights, and severity assumptions change how findings should be interpreted and prioritized.

## Downloadable reports

The Downloads tab provides:

- Enterprise Data Strategy Brief markdown.
- Metadata Lint Report markdown.
- Remediation Plan markdown.
- Remediation Backlog JSON.

These artifacts are generated by the existing linting, analysis, remediation planning, and report modules.

## Current limitations

- The UI is intentionally lightweight and does not try to replace the CLI.
- Domo-style metadata is the first reference implementation.
- Sample data is synthetic.
- No real Domo credentials are required.
- No real Domo API calls are made.
- No source platform is modified.
- Streamlit rendering is not deeply unit-tested; tests focus on helper functions and artifact generation.

- Visual trend views for repeated inventory snapshots.
- Optional connectors for Domo, Snowflake, Power BI, Tableau, Looker, dbt, Collibra, Alation, Atlan, or other platforms.
